Conectar GitHub
Implemente con cada envío a su rama vinculada.
Conecta tuGitHubCuenta para Renderizar y comenzar a implementar aplicaciones y sitios usando cualquier repositorio al que tenga acceso. Render reimplementa automáticamente su proyecto con cada envío a su rama vinculada (puedeDeshabilitar esto).

Render también puede poner en marcha uninstancia de vista previade su proyecto con cada solicitud de extracción abierta para ayudarlo a validar los cambios.

Configuración
Cuando creas tu primer servicio en elPanel de renderizadoSe le solicitará que conecte su proveedor de Git:

Conexión de Git

Haz clic en GitHub . Esto te redirigirá a GitHub para que puedas autorizar a Render a acceder a tus repositorios.

Luego serás redirigido nuevamente al Panel de Render, que ahora muestra una lista de tus repositorios de GitHub:

Lista de repositorios de GitHub en el panel de renderizado

¡Has vinculado tu cuenta de GitHub correctamente! Al crear un nuevo servicio, selecciona cualquier repositorio disponible y haz clic en " Conectar" . Luego, completa el resto del proceso de creación del servicio.

Vistas previas de solicitudes de extracción
Render puede crear e implementar automáticamente una instancia de vista previa de su servicio para cada solicitud de extracción que se abra en su proyecto.

Para más detalles, véaseVistas previas del servicio.

Submódulos de Git
Si su repositorio define un .gitmodulesarchivo en su raíz, Render lo lee automáticamente y clona todos los especificados.Submódulos de Gitcomo parte del proceso de construcción de su servicio.

Si su .gitmodulesarchivo incluye submódulos privados , Render puede clonarlos solo si su cuenta de GitHub vinculada tiene acceso al repositorio privado correspondiente.

Iniciar sesión con GitHub
Además de implementar proyectos desde GitHub, puedes usar tu cuenta de GitHub para iniciar sesión enPanel de renderizadoSi ya tienes una cuenta de Render que coincide con la dirección de correo electrónico principal de tu cuenta de GitHub, Render te iniciará sesión en esa cuenta automáticamente.

Obtenga más información sobregestión de métodos de inicio de sesión.

Solución de problemas
Si tus implementaciones de GitHub no funcionan como se espera, esto podría deberse a una configuración incorrecta de la aplicación de GitHub de Render. Por ejemplo, podría estar configurada para un conjunto de repositorios incorrecto, o un repositorio que antes era público podría haberse convertido en privado.

Corrección de los permisos de la aplicación GitHub
Visitagithub.com/apps/render/instalaciones/nuevoDesde aquí, puedes instalar la aplicación en una nueva organización o configurar una instalación existente:

Configurar GitHub

Desde aquí, puedes consultar la sección de acceso al repositorio para asegurarte de que tu repositorio esté incluido.

Seleccionar repositorios de GitHub conectados

Problemas específicos del equipo
Si el creador de un servicio de Render pierde el acceso al repositorio de GitHub conectado a ese servicio, puede interrumpir las implementaciones de ese servicio. Puede actualizar las credenciales de Git utilizadas para implementar un servicio desde la página de configuración del servicio en el...Panel de renderizado:

Credenciales de Git

Antes de realizar este cambio, asegúrese de que las nuevas credenciales tengan acceso al repositorio Git del servicio.
