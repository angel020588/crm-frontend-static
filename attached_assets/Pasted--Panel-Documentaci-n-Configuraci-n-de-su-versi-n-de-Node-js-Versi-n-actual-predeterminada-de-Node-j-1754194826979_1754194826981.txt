Configuración de su versión de Node.js
Versión actual predeterminada de Node.js
22.16.0

Los servicios creados antes del 12/06/2025 tienen una versión predeterminada diferente.Vea abajo.

Establezca una versión diferente de Node.js de cualquiera de las siguientes maneras (en orden descendente de precedencia):

Establezca la NODE_VERSIONvariable de entorno para su servicio en elPanel de renderizado:

Configuración de la variable de entorno NODE_VERSION

Añade un archivo llamado .node-versiona la raíz de tu repositorio. Este archivo contiene una sola línea con la versión a usar:

18.18.0
Añade un archivo llamado .nvmrca la raíz de tu repositorio. Este archivo usa el mismo formato que .node-version.

Especifique un rango de versiones de Node.js en su package.jsonarchivo, en elenginespropiedad:

"engines": {
  "node": ">=18.18.0 <19.0.0"
}
Si no hay un package.jsonarchivo en el directorio raíz de su repositorio, Render usa el primer package.jsonarchivo que encuentra en un subdirectorio.

Incluya siempre un límite superior en su rango de versiones.

Un rango ilimitado (como >=18) siempre se resuelve en ellatestliberarde Node.js, cuya versión principal se actualiza con el tiempo. Esto podría generar comportamientos inesperados o incompatibilidades con la versión de desarrollo.

Puede especificar un número de versión semántica (como 18.18.0) o un alias (como lts).

Render utiliza elnode-version-aliasMódulo para resolver alias de versiones ysemverrangos.
